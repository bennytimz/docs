description: "What your agent needs to operate onchain, and the tools to get there"

Onchain rails give your agent programmable payments, global reach, instant settlement, and the ability to transact with other agents without intermediaries. This stack covers what's required: wallets, identity, verification, actions, and the ecosystem of agent-native apps.

## Core concepts

<CardGroup cols={2}>
  <Card title="Wallets" icon="wallet" href="/base-agents/core-concepts/wallets">
    Wallets are the first step to getting your agent onchain. They give your agent the ability to hold funds and execute transactions.
  </Card>

  <Card title="Identity and auth" icon="fingerprint" href="/base-agents/core-concepts/identity-verification-auth">
    Your agent needs to be discoverable. Registration puts it onchain so other agents and services can find and verify it.
  </Card>

  <Card title="Onchain actions" icon="bolt" href="/base-agents/core-concepts/onchain-capabilities">
    Agents can utilize X402 for payments and execute smart contract interactions. Skills define the tools your agent can call including swaps, trades, balance checks, and protocol interactions.
  </Card>

  <Card title="Agent apps" icon="grid-2" href="/base-agents/core-concepts/agent-apps">
    Agent apps are services built with agents as the primary user. Learn how to build for agents and make your app discoverable so agents can find and interact with it.
  </Card>

  <Card title="Frameworks" icon="cubes" href="/base-agents/core-concepts/agent-frameworks">
    Agent SDK, OpenClaw, and BANKR managed service - options for building and running agent logic on Base.
  </Card>
</CardGroup>
